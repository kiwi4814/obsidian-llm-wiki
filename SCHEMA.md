# SCHEMA.md：Obsidian LLM Wiki 笔记结构规范

本文档定义本 vault 中结构化知识页的格式。Hermes 创建或更新 `20-知识库/` 下的笔记时必须遵守。

## 1. 三层知识流

```text
00-收集箱 → 20-知识库 → 30-输出
```

- `00-收集箱`：原始输入，不追求结构完美；
- `20-知识库`：长期知识，要求结构化；
- `30-输出`：面向阅读、发布、复用的成果。

## 2. 笔记类型

允许的 `类型`：

- 原始输入
- 每日记录
- 概念
- 实体
- 主题
- 对比
- 问答
- 项目
- 输出
- 模板
- 归档

## 3. 状态

允许的 `状态`：

- 待整理
- 待确认
- 草稿
- 可用
- 成熟
- 归档

## 4. 通用 frontmatter

所有结构化知识页建议使用：

```yaml
---
标题: 笔记标题
类型: 概念
状态: 可用
tags:
  - 示例
来源:
  - 00-收集箱/Telegram/示例.md
创建时间: YYYY-MM-DD
更新时间: YYYY-MM-DD
摘要: 一句话说明这篇笔记讲什么。
别名: []
相关: []
保护: false
---
```

## 5. 文件命名规则

- 使用中文文件名；
- 文件名短、清晰、稳定；
- 不使用无意义编号作为主标题；
- 避免特殊字符：`# | ^ : %% [[ ]]`；
- 同一概念只保留一个主页面，别名写入 frontmatter。

## 6. 概念页结构

路径：

```text
20-知识库/概念/概念名.md
```

结构：

```markdown
# 概念名

## 一句话定义

## 核心解释

## 使用场景

## 常见误区

## 相关笔记

## 来源
```

## 7. 实体页结构

路径：

```text
20-知识库/实体/实体名.md
```

结构：

```markdown
# 实体名

## 简介

## 关键特征

## 评价

## 相关事件或记录

## 与其他实体的关系

## 待确认

## 来源
```

## 8. 主题页结构

路径：

```text
20-知识库/主题/主题名.md
```

结构：

```markdown
# 主题名

## 主题概述

## 核心问题

## 当前结论

## 子主题

## 相关概念

## 相关实体

## 待处理问题

## 来源
```

## 9. 对比页结构

路径：

```text
20-知识库/对比/A 与 B 对比.md
```

结构：

```markdown
# A 与 B 对比

## 结论摘要

## 对比维度

| 维度 | A | B | 备注 |
|---|---|---|---|

## 适用场景

## 风险与限制

## 待确认

## 来源
```

## 10. 问答沉淀结构

路径：

```text
20-知识库/问答沉淀/问题标题.md
```

结构：

```markdown
# 问题标题

## 问题

## 简短回答

## 详细分析

## 依据的笔记

## 可继续追问

## 更新时间
```

## 11. 链接规则

- 首次出现重要概念时使用 `[[概念名]]`；
- 引用实体时使用 `[[实体名]]`；
- 不为每个普通词都加链接；
- 链接应该帮助导航，不应该制造噪音。

## 12. 标签规则

标签要少而稳定。

推荐基础标签：

```text
AI
知识管理
Obsidian
Hermes
LLM-Wiki
动漫
字幕组
读书
写作
工具
调研
项目
生活
```

不要随意创造大量相近标签。

## 13. 来源规则

重要结论必须能追溯来源。

来源可以是：

- `00-收集箱/Telegram/...`
- `00-收集箱/网页剪藏/...`
- `00-收集箱/读书摘录/...`
- 其他知识页
- 外部链接

## 14. 冲突规则

如果不同来源说法冲突，不要直接合并成一个看似确定的结论。

应写：

```markdown
## 冲突与待确认

- 来源 A 认为：...
- 来源 B 认为：...
- 当前暂定：...
- 待确认：...
```

## 15. 质量标准

一篇合格知识页应该满足：

- 标题清晰；
- frontmatter 完整；
- 有一句话摘要；
- 有来源；
- 有相关链接；
- 不确定内容有标注；
- 不混入大量原始未整理材料。
