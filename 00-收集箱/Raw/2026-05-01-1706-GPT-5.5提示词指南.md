# GPT-5.5 提示词指南

别再写长提示词了！

OpenAI 官方建议：少写流程，多写目标

![[附件/图片/2026-05-01-GPT-5.5提示词指南.jpg]]

## 模型速览

- 发布周期：距 GPT-5.4 仅 6 周
- API 价格：输入 $5/百万 Token，输出 $30/百万 Token
- 上下文长度：约 105 万 Token
- 开放对象：Plus / Pro / Business / Enterprise

## 核心变化：新旧写法对比

**旧写法**：步骤繁琐（先查→再比→分析→总结→输出）→ 太啰嗦、像噪音

**新写法**：目标清楚、成功标准清楚、限制条件清楚 → 让模型自己规划路径

原则：越短越好，告诉它"要什么"，而不是"怎么做"

## 4 个关键变化

1. **越短越好**：写清目标、成功标准、限制条件；别把步骤全写死（长流程会缩小模型搜索空间）
2. **给 AI 装上"性格"和"刹车"**：
   - 性格：定义语气 + 合作方式
   - 刹车：设置检索预算 + 停止条件（能回答就停，别无限搜索；缺关键事实再继续搜）
3. **先给用户一句开场白**：长任务开始前，先说"收到，我先处理这个。"（体感会更快）
4. **创意写作也要分清事实和发挥**：产品数据/客户信息/路线图必须有出处；没有数据就用占位符

## 推荐模板（6 模块）

每个模块尽量简短，只在会改变模型行为时加细节：

1. 角色
2. 性格
3. 目标
4. 成功标准
5. 限制条件
6. 停止规则

## 提示词工程的演变

- GPT-4 时代："提示词工程"
- GPT-5.5 时代："少说多做"
- 以前是"人适应模型的局限"，现在是"模型更能适应人的模糊表达"
- 警示：还在用两年前模板的团队，可能该重写提示词了
