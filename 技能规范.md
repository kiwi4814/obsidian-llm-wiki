# 技能规范

本文档规定 Hermes 为本 vault 创建或更新 skills 时的格式。

## 语言规则

- skill 的目录名和 `name` 可以使用英文短横线，例如 `obsidian-capture`。
- 标题、说明、步骤、错误提示、Telegram 回复、日志、提交信息必须使用中文。
- 命令、环境变量、路径、API 字段名可以保留英文。

## 每个 skill 必须包含

- 使用场景
- 触发方式
- 操作范围
- 执行步骤
- 安全规则
- 输出格式
- 验证方式

## 不要做

- 不创建万能大 skill。
- 不把密钥写进 skill。
- 不在 skill 中硬编码用户 token。
- 不让 skill 默认删除原始输入。
